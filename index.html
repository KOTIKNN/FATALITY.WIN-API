Using lua scripts

1. Save the Lua script in the same folder as workspace\Fatality\lua. The file extension should be ".lua".

2. Go to the "MISC" tab. Here you'll see a list of all installed lua scripts.

3. Select the script you want to load and click "Load". "Unload" disables the script. "Reload Active Script" restarts active scripts.

Getting started

Lua scripts let you extend the cheat in various ways and are a great way to automate parts of the cheat or add that one unique feature you always wanted. To get started with Lua scripting, you'll need a suitable text editor. We recommend downloading VS Code or Sublime Text, but Notepad++ or even the built-in Microsoft Notepad will work in a pinch. If you're unfamiliar with the Lua programming language, Lua in 5 minutes is a great guide to get started.

Examples

These example show how to use certain parts of the FATALITY.WIN API and highlight best practices.

ui.add_element

Functions for interfacing with the FATALITY menu.

Fatality.ui.add_element(tabName: string, panelName: string, item: table, other: string)

Adds a UI element to a specified tab and panel in the FATALITY menu.

Argument Type Description
tabName string Name of the tab where the element will be added
panelName string Name of the panel (ScrollingFrame) within the tab
item table Configuration table for the UI element, including type and required parameters
other string Additional parameters (context-specific)

This function allows you to dynamically add UI elements like checkboxes, comboboxes, sliders, etc., to the FATALITY menu. The item table must include a type field (e.g., "checkbox", "combobox", "slider") and the required parameters defined in elementConfig. Supported element types include: checkbox, combobox, colorpicker, slider, bind, textentry.

Examples of adding different UI elements:

Checkbox Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "checkbox",
    text = "Snipline",
    var = "Snipline",
    colpicker = "SniplineCol", -- or colpicker2 = "SniplineCol2"
    bind = "SniplineBind" -- or bind2 = "SniplineBind2"
})

Combobox Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "combobox",
    text = "Custom ComboBox",
    var = "ComboBox",
    options = {"Option1", "Option2", "Option3"},
    moresave = true,
    colpicker = "ComboBoxCol" -- or bind = "ComboBoxBind"
})

Colorpicker Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "colorpicker",
    text = "Custom ColorPicker",
    var = "ColorPicker",
    colpicker = "ColorPickerCol" -- or bind = "ColorPickerBind"
})

Slider Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "slider",
    text = "Custom Slider",
    var = "Slider",
    min = 0,
    max = 100,
    dec = 1,
    colpicker = "SliderCol" -- or bind = "SliderBind"
})

Bind Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "bind",
    text = "Custom Bind",
    var = "Bind",
    colpicker = "BindCol" -- or bind = "BindBind"
})

Textentry Example:

Fatality.ui.add_element("Aimbot", "Visual", {
    type = "textentry",
    text = "Custom TextEntry",
    var = "TextEntry",
    colpicker = "TextEntryCol" -- or bind = "TextEntryBind"
})

ui.upd_element

Fatality.ui.upd_element(varName: string, item: table)

Updates an existing UI element in the FATALITY menu.

Argument Type Description
varName string Name of the variable associated with the UI element
item table Configuration table with updated properties (var, bind, color, or options)

This function updates the properties of an existing UI element, such as its value, bind, color, or options. It validates the input and updates the corresponding configuration in Fatality.config.

Examples of updating UI elements:

Fatality.ui.upd_element("DistanceEntity", {
    var = 10000
})
Fatality.ui.upd_element("enableAimbot", {
    var = false
})
Fatality.ui.upd_element("enableAimbotBind", {
    bind = "MouseButton1"
})
Fatality.ui.upd_element("FreeCameraBind", {
    bind = "Space"
})
Fatality.ui.upd_element("ShowFOVCol", {
    color = Color3.new(0.5, 0.5, 0.5), alpha = 0.7
})
Fatality.ui.upd_element("TargetPriority", {
    options = {"NewOption1", "NewOption2", "NewOption3", "NewOption4", "NewOption5", "NewOption6"}
})
Fatality.ui.upd_element("ParticleCustom", {
    var = "я гей"
})

ui.add_pal

Fatality.ui.add_pal(tabName: string, sectionName: string, palNumber: number, panelName: string, height: number)

Adds a new panel to a specified column in a section of the FATALITY menu.

Argument Type Description
tabName string Name of the tab where the panel will be added
sectionName string Name of the section within the tab
palNumber number Column number (1, 2, or 3) where the panel will be placed
panelName string Name of the new panel
height number Height of the panel (optional, defaults to 180)

This function creates a new scrolling panel with a title in the specified column of a section within a tab. It adjusts the layout order, adds a title frame, and creates a scrolling frame for UI elements.

Example:

Fatality.ui.add_pal("Aimbot", "Aimbot", 2, "General", 200)

ui.update

Fatality.ui.update()

Refreshes the display of all UI elements in the FATALITY menu.

This function iterates through all UI element collections (ComboBoxes, Sliders, CheckBoxes, ColorPickers, Binds, and TextEntries) and calls their respective update functions to refresh their display.

Example:

Fatality.ui.update()

ui.get_all

Fatality.ui.get_all()

Retrieves and logs information about all UI elements in the FATALITY menu.

This function collects data about all tabs, panels, and their associated UI elements, including their types, text, variables, and additional properties.

Example:

Fatality.ui.get_all()

ui.is_menu_open

Fatality.ui.is_menu_open()

Checks if the main FATALITY menu is currently open.

This function returns a boolean indicating whether the main menu frame is visible.

Example:

Fatality.RunService.Heartbeat:Connect(function()
    if Fatality.ui.is_menu_open() then
        print("FATALITY menu is open")
    else
        print("FATALITY menu is closed")
    end
end)

client_rgba

Fatality.client_rgba(varName: string)

Retrieves the RGBA values of a color stored in Fatality.config.colors for a given variable name.

Argument Type Description
varName string Name of the variable associated with the color

This function converts a Color3 value and its associated alpha value from Fatality.config.colors into RGBA values (0-255).

client_color3

Fatality.client_color3(r: number, g: number, b: number, a: number)

Converts RGBA values to a Color3 object and an alpha value for use in the FATALITY system.

Argument Type Description
r number Red component (0-255)
g number Green component (0-255)
b number Blue component (0-255)
a number Alpha component (0-255, optional, defaults to 255)

This function takes RGBA values, clamps them to valid ranges, and converts them into a Color3 object with an inverted alpha value (where 0 is opaque and 1 is transparent).

Example Usage:

local r, g, b, a = Fatality.client_rgba("EspCol")
Fatality.client_log(string.format("RGBA EspCol: %d, %d, %d, %d", r, g, b, a))

local color3, alpha = Fatality.client_color3(r, g, b, a)
Fatality.client_log(string.format("Color3: R=%.2f, G=%.2f, B=%.2f, Alpha=%.2f", color3.R, color3.G, color3.B, alpha))
